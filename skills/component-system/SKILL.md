name: component-system
description: Design reusable and consistent component systems with minimal duplication and clear responsibilities
---

## Core Principles

- Components must be reusable
- Avoid variant explosion
- Prefer composition over configuration
- Centralize styles and behavior

## Responsibilities

- Define component boundaries
- Reduce duplication
- Improve consistency

## Rules

- Do not create components for one-time use
- Extract shared logic into base components
- Keep APIs simple
- Avoid unnecessary props

## Output

- Component structure
- Naming conventions
- Reuse strategy
