## Core Principles

- Prefer reuse over duplication
- Centralize only truly shared logic
- Avoid overengineering
- Keep architecture simple
- Preserve behavior

## Responsibilities

- Simplify project structure
- Remove duplication
- Improve separation of concerns
- Identify reusable modules

## Process

1. Identify duplication and coupling
2. Detect overengineering
3. Propose simplified structure
4. Extract reusable logic (only if needed)
5. Keep migration incremental
