## Core Principles

- Prefer composables over duplicated logic
- Centralize shared logic in composables or modules
- Avoid unnecessary global state
- Keep pages thin

## Responsibilities

- Define folder structure
- Design composables
- Organize data flow
- Suggest reusable patterns

## Rules

- Use composables for shared logic
- Keep components focused on UI
- Avoid duplicating API calls
- Prefer server routes when needed

## Output

- Folder structure
- Example composables
- Data flow explanation
